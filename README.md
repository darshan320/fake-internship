# Network Congestion Prediction using Ensemble Learning

## Project Description
This project demonstrates the process of generating synthetic network performance data, training individual machine learning models (K-Nearest Neighbors, Decision Tree, Logistic Regression), and then combining them into an ensemble Voting Classifier to predict network congestion. The goal is to build a robust model that can identify periods of congestion based on various network metrics.

## Table of Contents
1. [Setup and Data Generation](#setup-and-data-generation)
2. [Bootstrap Sampling](#bootstrap-sampling)
3. [Base Model Training](#base-model-training)
4. [Ensemble Model (Voting Classifier)](#ensemble-model-voting-classifier)
5. [Model Evaluation and Comparison](#model-evaluation-and-comparison)

## 1. Setup and Data Generation
This section covers the initial setup, including importing necessary libraries and generating a synthetic dataset that simulates network performance metrics and congestion status.

### Data Fields:
- `Timestamp`: Datetime when the data was recorded.
- `Latency_ms`: Network latency in milliseconds.
- `Packet_Loss_Percentage`: Percentage of packets lost.
- `Bandwidth_Usage_Mbps`: Bandwidth usage in Megabits per second.
- `User_Count`: Number of active users.
- `Is_Congested`: Binary target variable (1 if congested, 0 if not).

The data generation includes a correlation logic to ensure that high latency, high packet loss, and high user count are indicative of network congestion.

**Output:** A CSV file named `network_congestion_data.csv` will be created, and the first few rows of the generated data, along with the distribution of the `Is_Congested` class, will be displayed.

## 2. Bootstrap Sampling
To introduce diversity and improve the robustness of the ensemble model, bootstrap sampling is used. Three distinct bootstrap samples are created from the original dataset. Each base model in the ensemble will be trained on a different bootstrap sample.

**Process:**
- The original dataset is loaded.
- Features (`X`) and target (`y`) are defined.
- `n_subsets` (e.g., 3) bootstrap samples are generated by resampling with replacement.

**Output:** The shapes of the subsamples and the value counts for the target variable (`Is_Congested`) in each bootstrap sample are printed.

## 3. Base Model Training
Three different machine learning models are chosen as base learners for the ensemble:
- **K-Nearest Neighbors (KNN)**
- **Decision Tree Classifier**
- **Logistic Regression**

Each of these models is trained on a separate bootstrap sample to ensure they learn slightly different patterns from the data.

**Output:** Confirmation messages for the training of each base model.

## 4. Ensemble Model (Voting Classifier)
An `sklearn.ensemble.VotingClassifier` is used to combine the predictions of the three base models. This project uses 'hard' voting, meaning the final prediction is determined by the majority vote of the base classifiers.

**Process:**
- The trained KNN, Decision Tree, and Logistic Regression models are passed to the `VotingClassifier`.
- The ensemble model is then trained on the original full dataset (or a dedicated training set if available).

**Output:** A confirmation message that the ensemble model has been created and trained.

## 5. Model Evaluation and Comparison
This section evaluates the performance of the ensemble model and compares its accuracy with that of the individual base models.

### Ensemble Model Evaluation:
- **Classification Report:** Provides precision, recall, f1-score, and support for each class.
- **Accuracy Score:** Overall accuracy of the ensemble model.
- **Confusion Matrix:** A visual representation of the model's predictions versus actual values, showing true positives, true negatives, false positives, and false negatives.

### Comparison with Base Models:
- The accuracy of each individual base model (KNN, Decision Tree, Logistic Regression) is calculated by predicting on the entire dataset.
- A bar plot visualizes the accuracies of the individual models against the ensemble model, allowing for a clear comparison of performance.

**Output:**
- Classification report, accuracy score, and a confusion matrix plot for the ensemble model.
- A bar chart comparing the accuracies of all models (KNN, Decision Tree, Logistic Regression, Ensemble).

## Results Overview
The ensemble model typically achieves higher or comparable performance compared to individual models due to its ability to leverage the strengths of different algorithms and reduce variance. The plots and reports provide insights into which models perform best and where the ensemble excels.
